◆ IsLineCurrentlyRecordable()

bool HearThis.Script.Project.IsLineCurrentlyRecordable ( int book,
int chapterNumber1Based,
int lineNo0Based )

Strictly speaking, skipped lines are not recordable, but we leave the button semi-enabled (grayed out and doesn't actually allow recording, but it does give the user a message if clicked), so we want to treat them as "recordable" if skipping is the only thing standing in the way. Similarly, if we're in overview mode, nothing can currently be recorded, but we want to show things that could be if we were in the right character as recordable.

Parameters
bookScripture book number, 0-based
chapterNumber1Based1-based (0 represents the introduction)
lineNo0Based0-based (does not necessarily/typically correspond to verse numbers). When called for a project that uses a filtered set of blocks, this should be the unfiltered block number.
Returns
A value indicating whether the specified book, chapter and line refer to a location that a) exists in the script and b) is not filtered out for the currently selected Actor/Character (if any).
